Mis Primeras Líneas.

Como primer post técnico me pareció buena idea hacer una guía a la vex que cuento mi experencia a la hora de hacer un blog y subirlo a la web.   1. Tener instalado Go

  3. Cuenta en GitHub

Gestores de Contenido

Los gestores de contenido son entornos que proporcionan una serie de plantillas, herramientas y códigos para acelerar el desarrollo de construcción de una página. A día de hoy poca monta un sitio web desde cero.

Y a no ser que quieras aprender HTML,CSS y JavaScript vas a necesitar un Framework que te ayude a crear una página web. Aún así si quieres una mayor personalización de tu portal vas a tener que aprender algo de estos lenguajes.

Cuando comencé mi busqueda de como crear un blog …..,gestores de contenido o framworks web en Google, lo primero que te aparecera es Wordpress, la herramienta por exelecia que todo el mundo te recomienda.

Sin embargo soy una persona que me gusta informarme y saber el funcionamiento de algo antes de embarcarme en un proyecto.

Lo primero es que existe una diferencia entre .org y .com

Lo segundo es que esta basado en PHP por lo que necesitas una base de datos en tu servidor y la seguridad que ello conlleva.

Y lo tercero que es la manera más fácil de hacer una web Dinámica. Yo no tenía ni idea de que era eso, ahí es cuando descubro lo que es una web estática.

Webs Estáticas

Es la plataforma en la que estan basadas la mayoría de webs.Pero yo quería algo más liviano, no quería tener una base de datos,no quería aprender PHP.

Conociendo a HUGO.

Como no tengo mucho interés en aprender PHP y Go es un leguaje que me encantó desde que lo conocí, me decante por el Framwork basado en el HUGO. Gestor paginas web estáticas

Una vez instalado HUGO nos dirigimos a terminal y comprobamos la correcta instalación mediante el comando hugo versionque nos dice la versión que estamos utilizando.

Para empezar un nuevo proyecto se hace uso del comando hugo new siteseguido del nombre del proyecto.

#No se debe escribir :~$ 
~$ hugo version
#Algo asi debera salir el +extended es por la versión extendida
hugo v0.92.1+extended linux/amd64
#Creamos un proyecto, miweb sera el ejemplo
~$ hugo new site miweb
#Resultado
Congratulations! Your new Hugo site is created in .....   
#Vamos al nuevo directorio creado 
~$ cd miweb
#Comando para ver contenido del directorio; en Linux "ls", en Windows "dir"
~/miweb$ ls
#HUGO ya nos ha creado un serie de directorios
archetypes  config.toml  content  data  layouts  static  themes

Una breve descripción de que es que.

Directorios Descripción
archetypes Carpeta contiene un archivo con la configuración predeterminada de los contenidos de tu web
config.toml Este archivo contiene la configuración de la web
content Carpeta donde se almacena el contenido de tu web
data Lugar donde se almacenan archivos de configuración de tu portal
layout Se encuentran las plantillas html que muestran las vistas de tu web
static Donde se almacena todo el contenido estático de tu web: Imagenes, CSS, JavaScrip, etc
themes Donde se guardan los temas o plantillas para generar nuestra web. Se pueden ver en la paguina de HUGO

Más adelante el propio programa creará dos carpetas más

Directorios Descripción
resource
public Aquí se crean por defecto los archivos estáticos de nuestra web

Me Encanta el Tema

Ahora toca elegir el tema que definira el aspecto de nuestra web, puedes elegir la plantilla que más te guste en https://themes.gohugo.io/.

Se pueden descargar de dos formas:

  1. Dirigiendote al repositorio del tema, copias su contenido en download ZIP y descomprimes los archivos en la carpeta themes .

  2. En la pantalla de comandos, en el directorio de nuestro proyecto introduces git init y acto seguido git submodule add más <URL_Repo_Tema> espacio <Carpeta_Themes>.

~/miweb$ git init    
 
~/miweb$ git submodule add https://github.com/dillonzq/LoveIt.git themes/loveit

En mi caso elegí el tema LoveIt, sin embargo si vas ha seguir esta opción tengo una mala noticia. Por lo visto este tema se encuentra abandonado y aunque siga existiendo el repositorio por no se encuentra mantenido.

Pero no todo son malas noticias, un usuario de Github a ramificado el proyecto para mantenrlo el mismo, añadiendo alguna que otra mejora. Puedes encontrar esta plantilla bajo el nombre Ublogger.

De todas formas estos dos temas, LoveIt y Ublogger se configuran de la misma forma. Posiblemente en un futuro no muy lejano me cambie a esta nueva versión pero de eso hablaré en otro post.

Por último vamos a ir a la carpeta themes y copiamos el contenido de layout en nuestra carpeta principal del mismo nombre.

git submodule

Si has hecho un git submodule tendras la información del tema dentro de subcarpeta con el nombre de este y esto puede ser conflictivo.

La manera más rápida de solucionar el problema es colocando los archivos de la subcarpeta en la carpeta themes.

Ahora para empezar a crear contenido es tan fácil como introducir hugo new más la <Dirección/Nombre_Artículot.md>. La extensión debe ser .md el formato de Markdown.

#Para crear nuestro primer artículo
~/miweb$ hugo new posts/prueba.md

Con esto hemos creado dentro de la carpeta content una archivo llamado prueba dentro de una subcarperta llamada post. Abrimos prueba.md con cualquier editor de texto ,empezamos a escribir y guardamos.

Para ver los resultados introducimos el comando hugo server , nos dirigimos a nuestro navegador a http://localhost:1313/ y listo.

~/miweb$ hugo server

Después de esto se crearán dos carpetas más en nuestro proyecto resourse y public

Y ahora, ¿Donde me hospedo?

En la documentación de HUGO te recominda diferentes hosting gratuitos compatibles. De todas las opciones me decanto por Github Pages con la idea de en algún futuro motar mi propio servidor privado.

Normas de Uso Github Pages

Usos prohibidos
Las Páginas de GitHub no se deben usar para realizar transacciones que impliquen el envío de información confidencial como contraseñas o números de tarjeta de crédito.

Además, su uso de Páginas de GitHub está sujeto a los Términos de servicio de GitHub , incluidas las restricciones sobre esquemas para hacerse rico rápidamente, contenido sexualmente obsceno y contenido o actividad violento o amenazante.

Límites de uso

Los sitios de Páginas de GitHub publicados no pueden tener más de 1 GB.

Los sitios de Páginas de GitHub tienen un límite de ancho de banda flexible de 100 GB por mes.

Los sitios de Páginas de GitHub tienen un límite flexible de 10 compilaciones por hora.

Lo primero que se debe hacer es crear un repositorio con el nombre Tu_Usuario.github.io , el

Añade un nuevo archivo a tu repositorio con la siguiente ruta .github/workflows/gh-pages.yml. Esto creará un archivo gh-pages.yml en la carpeta .github/workflows/.

En este archivo copia el siguiente código:

name: github pages

on:
  push:
    branches:
      - main  # Set a branch to deploy
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-20.04
    steps:
      - uses: actions/checkout@v2
        with:
          submodules: true  # Fetch Hugo themes (true OR recursive)
          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
          # extended: true

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        if: github.ref == 'refs/heads/main'
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

En total nuestro repositorio debe contener tres archivos:

  • el archivo conf.tolm Cambiar Url por User.github.io.
  • Carpeta Docs donde se encuentran los contenidos estáticos.
  • Ejecutable github/workflows/gh-pages.yml.

Una vez ahí cambias la rama de Master o Main a Docs/. Con estos cambios ya puedes ver tus resultados introduciendo tu Url.

    Apuntando a Mi Host.

    Ya con el dominio dado de alta, nesesitamos dirigirlo a donde se encuentra hospedada nuestra página. Esto se consigue apuntando las direciones DNS de nuestro hosting*( Nuestro caso Github Pages )* al proveedor de nuestro dominio.

    Spoiler

    Las DNS que apuntan a tu pagina de github son

    185.199.108.153
    185.199.109.153
    185.199.110.153
    185.199.111.153

    La manera mas fácil de comprobarlo por consola es por medio del comando dig y la dirección github.io.

    
~$ dig UserGitHub.github.io    
#Entre más información se deberá ver 
;; ANSWER SECTION:

UserGitHub.github.io.		1531	IN	A	185.199.109.153
UserGitHub.github.io.		1531	IN	A	185.199.110.153
UserGitHub.github.io.		1531	IN	A	185.199.108.153

    Una vez hecho, entras en tu cuenta de namechep y te dirige a las opciones avanzadas de tu domino y añades las 4 DNS de Github, cambias en el nombre para apuntar a tu URL TuCuenta.github.io.

    Ya solo queda un último paso, volver a las opciones de nuestro repositorio en Github y añadir el nombre de nuestro dominio.

    Al hacer esto se crea un archivo llamado CNAME dentro de docs que redirigirá el tráfico de Github pages al dominio elegido.
